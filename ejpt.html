# NMAP PORT SCAN
nmap <TARGET_IP>
## Skip ping
nmap -Pn <TARGET_IP>
## Host discovery + saving into file
nmap -sn <TARGET_IP>/<SUB> > hosts.txt
nmap -sn -T4 <TARGET_IP>/<SUB> -oG - | awk '/Up$/{print $2}'
## Scan all ports
nmap -p- <TARGET_IP>
## Open ports scan + saving into file
nmap -Pn -sV -T4 -A -oN ports.txt -p- -iL hosts.txt --open
## Port 80 only scan
nmap -p 80 <TARGET_IP>
## Custom list of ports scan
nmap -p 80,445,3389,8080 <TARGET_IP>
## Custom ports range scan
nmap -p1-2000 <TARGET_IP>
## Fast mode & verbose scan
nmap -F <TARGET_IP> -v
## UDP scan
nmap -sU <TARGET_IP>
## Service scan
nmap -sV <TARGET_IP>
## Service + O.S. detection scan
sudo nmap -sV -O <TARGET_IP>
## Default Scripts scan
nmap -sC <TARGET_IP>
nmap -Pn -F -sV -O -sC <TARGET_IP>
## Aggressive scan
nmap -Pn -F -A <TARGET_IP>
## Timing (T0=slow ... T5=insanely fast) scan
nmap -Pn -F -T5 -sV -O -sC <TARGET_IP> -v
## Output scan
nmap -Pn -F -oN outputfile.txt <TARGET_IP> 
nmap -Pn -F -oX outputfile.xml <TARGET_IP> 
## Output to all formats
nmap -Pn -sV -sC -O -oA outputfile <TARGET_IP>

nmap -A -oA outputfile <TARGET_IP>

17. Methodology

A repeatable game plan for the eJPT exam — a black-box internal network assessment across multiple hosts with pivoting and web targets. Work host-by-host, keep notes and evidence as you go, and loop enumeration → exploitation → post-exploitation every time you gain a new foothold or reach a new subnet.

17.1 Pre-Engagement & Scoping

  • Confirm scope: in-scope subnets/IPs/URLs, rules of engagement, testing window, and what proof is required
  • Connect to the lab/VPN and confirm reachability to the target network
  • Set up note-taking (CherryTree / Obsidian) and an evidence folder per host

17.2 Host Discovery & Network Mapping

  • Identify your own IP and subnet (ip a, ip route) and infer the in-scope range
  • Sweep for live hosts (fping -ag, nmap -sn) and build a host list
  • Note dual-homed hosts and routes that could become pivot points

17.3 Port Scanning & Service Enumeration

  • Full TCP scan each host (nmap -p- -sV -sC), then targeted UDP (nmap -sU --top-ports 100)
  • Record every open port, service, and version into per-host notes
  • Enumerate each service deeply (SMB, FTP, SSH, HTTP, SMTP, SNMP, MySQL/MSSQL, RDP)

17.4 Web Application Enumeration

  • Fingerprint the stack (whatweb, Wappalyzer) and brute dirs/files/vhosts (gobuster, feroxbuster)
  • Identify any CMS (WordPress / Drupal / Joomla) and run targeted scanners (wpscan, droopescan)
  • Test for common flaws: SQLi, LFI/RFI, file upload, command injection, auth bypass

17.5 Vulnerability Assessment

  • Map service versions to known CVEs (searchsploit, NVD) and prioritise by exploitability
  • Run targeted checks (nmap --script vuln, Nessus if available)
  • Validate findings manually to weed out false positives

17.6 Exploitation & Initial Access

  • Match the exact version before running any public exploit — read the code first
  • Use Metasploit modules or manual exploits; catch the shell with a listener (nc -lvnp)
  • Brute-force weak service logins where in scope (hydra); reuse any discovered credentials

17.7 Post-Exploitation & Looting

  • Stabilise the shell (upgrade to a full TTY); identify OS, user, and privileges
  • Loot credentials, config files, hashes, SSH keys, and command history
  • Escalate privileges (Linux: sudo -l, SUID, kernel; Windows: services, tokens, misconfigs)

17.8 Pivoting & Lateral Movement

  • Identify new subnets reachable from the compromised host (ip route, arp -a)
  • Set up pivoting (Metasploit autoroute + SOCKS, or SSH tunneling) to reach internal hosts
  • Repeat enumeration → exploitation against the newly reachable targets

17.9 Reporting

  • Document each host: open ports, vulnerabilities, exploitation steps, and proof (screenshots, flags)
  • Capture commands and evidence as you go; note remediation recommendations
  • Answer the exam questions clearly, with reproduction steps for each finding
