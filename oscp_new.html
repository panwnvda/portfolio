7. Tunneling & Pivoting

7.1 SSH Forwarding

SSH tunnels let you reach internal networks through a compromised host.

# Local port forwarding — access internal_host:80 through pivot

# Dynamic port forwarding (SOCKS proxy)

# Configure proxychains: socks5 127.0.0.1 1080

# Remote port forwarding — expose your listener through pivot
ssh -R 9999:127.0.0.1:443 user@PIVOT_IP
# The pivot now forwards its port 9999 to your local 443

# Remote dynamic port forwarding (OpenSSH 7.6+)

# SOCKS proxy now runs on attacker side
# SSH on Windows (if OpenSSH is installed)

# Plink (PuTTY CLI) — for older Windows

11.5 Shell Upgrades

Upgrade a dumb shell to a fully interactive TTY.

# Python PTY upgrade
python3 -c 'import pty; pty.spawn("/bin/bash")'
# Press Ctrl+Z to background the shell
stty raw -echo; fg
# Press Enter twice
export TERM=xterm
export SHELL=/bin/bash
stty rows 40 columns 160
# Script upgrade (if no Python)
script -qc /bin/bash /dev/null
# Then same Ctrl+Z, stty raw -echo; fg

12. Report Writing

12.1 Exam Structure

The OSCP exam report must demonstrate a clear methodology and reproducible exploitation steps.

  • 3 standalone machines (20 points each, 60 total) + 1 AD set (3 machines, 40 points total)
  • Passing score: 70 points
  • Bonus points: 10 points for completing 80%+ of PEN-200 exercises and 30+ lab machines
  • Time limit: 23 hours 45 minutes for the exam, then 24 hours to submit the report
  • Metasploit restriction: Full Metasploit (meterpreter, exploit modules) allowed on only ONE machine

12.2 Screenshot Requirements

Every step must be documented with screenshots to prove exploitation.

  • Take a screenshot of every significant step: initial foothold, privilege escalation, flag collection
  • Screenshots must show the IP address and the command executed
  • Include whoami and hostname output in screenshots
  • Capture the proof.txt and local.txt files with the command used to read them
# Linux proof collection
hostname && whoami && ip a && cat /root/proof.txt
cat /home/*/local.txt 2>/dev/null
cat /root/proof.txt
# Windows proof collection
hostname
whoami
ipconfig
type C:\Users\Administrator\Desktop\proof.txt
type C:\Users\*\Desktop\local.txt

12.3 Report Format

Use the official OffSec report template. Structure each machine write-up consistently.

  • Executive Summary: High-level overview of findings
  • Per-machine sections:
    • Service enumeration results
    • Vulnerability identification
    • Exploitation steps (with screenshots)
    • Privilege escalation steps (with screenshots)
    • Proof screenshots (proof.txt / local.txt with IP visible)
  • Active Directory chain: Document the full attack path across all 3 AD machines
  • Remediation recommendations for each vulnerability

Tools for report writing:

  • OSCP Markdown Report Template (convert with Pandoc to PDF)
  • CherryTree or Obsidian for note-taking during the exam
  • Flameshot or Greenshot for screenshots
